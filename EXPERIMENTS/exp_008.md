# 实验 [EXP_008] — 双模型路由端到端评估（tree_v4 / tree_v4_noTE）

> 实验日期：2026-07-20
> 状态：已完成
> 关联决策：D22；承接 exp_005 遗留问题 1（双模型策略候选）与阶段 12 收尾任务书

## 目标

阶段 12 收尾把双模型策略落地为 App 路由（D22）：**醛/胺都已见于训练池 → tree_v4（TE 先验）；任一未见 → tree_v4_noTE（外推）**。本实验在上线前量化路由策略相对单一模型的收益与代价：

1. 路由 vs 单一 v3（现 App 默认）/ 单一 v4 / 单一 noTE；
2. 按"醛已见 × 胺已见"分桶，定位路由键在哪些查询类型上赚、哪些上亏；
3. 增设对照策略 `routed_strict`（仅醛胺**均**未见 → noTE，其余含"一新一熟"→ v4），检验规定路由键是否过严。

## 数据与方法

- 数据与特征缓存同 exp_005：`stage11_xy_cache.pkl`（3094 行 × 142 基特征）。
- 策略（同折训练，防泄漏铁律继承：TE 映射表只用训练折拟合）：

| 策略 | 参数 | TE | 频率降权 | 说明 |
|---|---|---|---|---|
| v3 | V3_PARAMS | ✗ | ✗ | 现 App 默认模型行为 |
| v4 | v4_mild | ✓ | ✓ | tree_v4 行为 |
| noTE | v4_mild | ✗ | ✓ | tree_v4_noTE 行为 |
| routed | — | — | — | 上线规则：双已见取 v4 预测，否则取 noTE |
| routed_strict | — | — | — | 对照：仅双未见取 noTE，其余取 v4 |

- **协议 A — 随机 KFold-5 × 3 种子（42/123/7）**：模拟 App 混合查询流。每折以训练折组成"已见单体池"，验证样本按路由键分发；合并 5 折算指标，报 3 种子均值±std。随机划分下验证集约 49% 双已见、48% 一新一熟、3% 双未见。
- **协议 B — LOGO 分桶（GroupKFold-10 按醛，无种子）**：验证集醛必未见（路由恒走 noTE），按"胺是否已见于训练折"分桶（胺已见 n=2123 / 胺未见 n=971），量化规定键在"醛新胺熟"查询上的代价。
- 脚本：`scripts/stage12_routing_eval.py`（逐折落盘可续跑；routed/routed_strict 由同折 v4/noTE 预测合成，不单独训练）；报告：`reports/routing_eval.json`。

## 读数

1. **路由的设计目标达成**：双已见桶 routed 0.9609 = v4，比现默认 v3 高 +0.007，MAE 0.0966 全体最优——池内查询拿到 TE 先验的全部收益。
2. **路由 vs 现默认 v3：全线不劣**。A 整体 +0.005、双已见 +0.007；B 整体 +0.031、胺已见 +0.018、胺未见 +0.056。相对现状是明确的帕累托改进 → 上线合理。
3. **规定键的代价在"一新一熟"混合查询**：A 混合桶 v4 0.9380 vs routed 0.9139（−0.024，逐种子稳定）；B 胺已见桶 v4 0.9172 vs routed 0.8862（−0.031）。单侧 TE 仍有强信号，规定键把这类查询错配给了 noTE。
4. **双未见场景 v4 vs noTE 无稳定赢家**：双留出网格协议 noTE +0.041（exp_005：0.6824 vs 0.6418），但协议 B 胺未见桶方向反转（v4 0.6739 vs noTE 0.6442），协议 A 双未见小桶（n≈92）三者打平。两协议折构成不同（频次均衡网格 vs 稀有胺伴随留出），差异在 fold 级噪声量级；**noTE 确定的优势是双未见 MAE**（双留出 0.316 vs 0.343、A 双未见桶 0.247 vs 0.346）与分数稳定性（v4 逐种子 std 0.048 最大）。
5. **对照 routed_strict 全面不劣于 routed**：双已见/双未见两桶与 routed 完全相同，混合桶 +0.024（A）/ +0.031（B 胺已见）；A 整体 0.9480 逼近单一 v4 的 0.9513。证据提示它是规定键的严格改进候选。
6. **单一 v4 在 A/B 两协议整体最高**——但它在最严的双留出网格（D21 金标准外推协议）0.6418 < noTE 0.6824 且 MAE 一致偏差；路由的核心价值正是保住双未见场景的稳健性，同时吃到池内收益。

## 结论

1. 双模型路由按任务书规定键上线（D22）：**对现默认 v3 是帕累托改进**，池内 +0.007、LOGO 态 +0.031，且双未见场景由 noTE 兜底（MAE 与稳定性更优）。
2. **遗留证据明确的改进候选**：routed_strict（仅双未见走 noTE）在全部已测桶上不劣于规定键，混合桶 +0.024~0.031；是否切换取决于 App 真实查询中"一新一熟"的占比——写入 D22 复盘点，待使用日志定型后复盘。
3. 池内模型维持 v4_int（含 Hadamard 交互）：v4_noint LOGO +0.015 在逐折噪声内（配对差均值 +0.010、差值 std 0.019，10 折仅 6 折胜），双留出 3 种子一致 −0.014，且交互项是「打分理由」配对级解释的来源（详细论证见 D22）。

## 复盘（2026-07-20 收官，D23）：routed_strict 评审通过与切换

### 评审结论

**切换：上线规则由 routed（任一未见 → noTE）改为 routed_strict（仅双未见 → noTE，其余含"一新一熟" → v4）。** D22 第一复盘点关闭，证据即本报告 proto_A / proto_B 数据，无需新增实验。

### 证据复核（本报告数据）

| 桶 | routed（原规则） | routed_strict（新规则） | 差值 | 判定 |

- **唯一回退**：协议 A 混合桶 MAE strict 0.1222 vs routed 0.1147（−0.0075），幅度约为 PR-AUC 收益的 1/3；项目主指标为 PR-AUC（推荐排序场景），且同型桶在协议 B 上 MAE 方向相反（strict 优）——回退不稳定、量级小，不否决切换，列为 D23 复盘点。
- **原暂缓理由不成立**：D22 以"一新一熟真实占比未知"暂缓切换；复核发现 routed_strict 在**每个**已测桶上都不劣于 routed，占比只影响整体收益大小、不改变方向，故评审不依赖查询日志。随机流量下混合查询约占 48%，是高频形态，收益可兑现。
