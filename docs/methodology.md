# 方法论

> 本项目的建模方法论和设计原理。
> 最后更新：2026-07-02

---

## 一、核心方法论原则

### 1. 任务形态：推荐而非分类

不问"这个组合能否成膜"（二分类），而问"在候选组合中，哪些最值得优先实验"（打分排序）。

**理由**：
- 二分类需要可靠的负样本，但发表偏差导致负样本不可信（MNAR）
- 推荐任务天然规避负样本问题——我们只需要相对排序
- 实验师真正需要的是"优先级建议"，而非绝对的是/否

### 2. 描述符优先于端到端学习

用手工/半自动化学描述符 + 树模型，而非 GNN 端到端学习。

**理由**：
- 样本量小（几千）→ 端到端学习过拟合
- 描述符注入领域知识（归纳偏置），提升小样本泛化
- 树模型 + SHAP 给出化学可解释洞察
- 旧项目 GNN 在小样本上仍只能学到表面相似性

### 3. 条件与结构同等重要

成膜是"结构 × 条件"的联合结果，不能脱离条件谈结构。

**理由**：
- 同一组合在不同条件下可能成膜/成粉（用户问题 3）
- 旧项目完全忽略条件（只用结构+规则）→ 信息损失
- 条件特征让模型学到"什么条件适合什么组合"

## 二、特征工程设计（阶段 3 核心）

### 解决"RF 像相似度排序"的关键

旧项目 RF 预测"都倾向小单体、像相似度排序"。根因：**描述符尺度未归一化**。

当所有特征都是绝对量（分子量、原子数、芳环数），小单体在特征空间天然更近 → 树模型退化成最近邻。

**解决**：所有描述符做**尺度归一化**：
- 按**反应位点数**归一：如"芳环数/反应位点数"而非"芳环数"
- 按**分子量**归一：如"功能基团占比"
- 用**比例型**特征：如"醛/胺反应位点比"、"两单体芳环数比"

### 特征体系（四类）

#### A. 分子结构描述符（复用旧项目 src/chemistry/）

| 来源 | 维度 | 内容 |
|---|---|---|
| linker_analyzer.py | 26 | 每单体12维（炔/芳/环/可旋转键/重原子/MW/反应位点/对称性）+ 配对4维 |
| conformer.py | 10 | 单体3D（扁平度/体积/sp3比/偶极矩/位阻等） |
| dimer.py | 10 | 二聚体3D（复合物扁平度/反应位点距离/平面夹角等） |
| hard_rules.py | 34 | 23维化学规则违反 + 11维F/CF3描述符 |
| RDKit 内置 | 待定 | MW、LogP、TPSA、H供体/受体等（全部归一化） |

⚠️ **全部做尺度归一化**，避免绝对量导致相似度退化。

#### B. 反应条件描述符（新项目增量，首次使用）

| 字段 | 编码方式 |
|---|---|
| temperature | 解析为数值（°C），归一化 |
| solvent | 分类（水/DCM/EtOAc/...），one-hot 或嵌入 |
| catalyst | 分类（乙酸/Sc(OTf)3/无/...） |
| synthesis_route | 分类（interfacial/solvothermal/...） |
| interface_type | 分类（liquid-liquid/solid-liquid/...） |
| stoichiometry | 解析为数值比，归一化 |

#### C. 交互特征

- 醛/胺反应位点比
- 两单体芳环数比
- 大小失配度
- 电子匹配度（吸电子/给电子）

#### D. 领域规则特征（继承旧项目 34 维）

见 DECISIONS D05。

## 三、评估方法

### 交叉验证：留一单体法（leave-one-monomer-out）

**普通 K-fold 不够**——会把同单体的不同配对分到训练集和验证集，虚高泛化。

**留一单体法**：每次留出某个单体的所有配对作为验证集。这真正测试"对未见过的单体的泛化能力"。

这是新项目相对旧项目的改进——旧项目用按 paper_id 分层 K-fold，没做单体级泛化测试。

### 指标

| 指标 | 用途 |
|---|---|
| PR-AUC | 主指标（与旧项目 GNN 可比，PR-AUC 0.784 为基线） |
| NDCG@K | 排序质量（推荐任务核心） |
| Hit@K | Top-K 中真正成膜的比例 |
| 化学多样性 | Top-K 推荐的结构多样性（Morgan Tanimoto） |
| 化学新颖性 | Top-K 中"前人未试过"的比例 |

### 消融实验（必做）

1. 有/无反应条件特征 → 验证 D03 的价值
2. 有/无规则特征 → 验证 D05
3. 归一化 vs 绝对量描述符 → 验证"根治相似度排序"
4. 树模型 vs GNN → 验证 D02

## 四、负样本策略

不强行造负样本（DECISIONS D01）。具体：
- 文献真实负样本：保留，标注为低置信度
- 化学规则合成负样本：保留（旧项目已验证有效），标注来源
- 粉末样本（0.8）：作为弱正样本，不当负样本（DECISIONS D04）
- 推荐任务下，负样本的作用是"提供对比锚点"，模型学的是相对排序

---

## 五、可解释性（SHAP）

每个推荐必须给出理由：
- 全局：哪些特征最重要？（SHAP summary plot）
- 局部：这个具体推荐为什么高分？（SHAP force plot）
- 化学洞察：SHAP 揭示的规律是否与化学知识一致？

**关键验证**：SHAP 能否独立学到旧项目硬规则编码的规律？如果能 → 规则特征可去掉；如果不能 → 规则特征必要。

## 六、避免旧项目的坑

| 旧项目教训 | 本项目对策 |
|---|---|
| 多任务梯度冲突 | 禁用多任务，纯单任务 |
| 小数据增广无效 | 不盲目增广，优先外推检测 |
| SMILES 缺失 29.5% | 阶段 2 重点补全 |
| 跨实验假配对 | 审计时检测，按实验组而非文献去重 |
| 条件从未使用 | 首次作为特征 |
| F-醛系统偏差 | 审计 F 相关样本，可能需特征工程改进 |
