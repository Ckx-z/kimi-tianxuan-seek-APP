- **现阶段工作流**：文献挖掘数据（约 1000 对醛胺反应对）→ 手工描述符特征 → XGBoost 树模型 → 双留出/LOGO 严格评估 → 双模型路由（池内 tree_v4 LOGO 0.878 / 双未见 tree_v4_noTE 0.6824）→ Gradio App 交付（成膜概率 + SHAP 中文理由 + 分子结构图 + 条件推荐 + Word 报告）。
- **"谁更科学"**：科学性的核心是评估纪律与场景匹配，不是模型名字。GNN 叙事上更"现代"，但本项目约 1000 对的小数据 + 双未见场景恰是 GNN 的先天短板；阶段 B 闭卷终审（exp_010/D25）：GNN 端到端 0.6317 < 描述符树 0.6824，embedding 迁移 −0.034 逐种子全负。论文中更站得住的写法是"树模型为骨干，GNN 作为严格对照被检验并证伪"——经过检验的选型与诚实的负结果，比硬上一个更差但更时髦的模型科学得多。

### 先说什么没偏

- 第①步（选定单体组）和第②步（DFT 初筛）本来就不是这个项目的事，谈不上偏。
- 第③步是项目主场：App 的交付形态（概率 + SHAP 中文理由 + 结构图 + 条件推荐 + Word 报告）正好就是这一步要的东西。

### 四处偏移（按严重度排）

**1. 验证错位——最大的偏移。项目从未用你的真实实验回测过。**

`实验ABCDEF.docx` 就是现成的、最有说服力的验证集，而它一直在我们视野之外。里面的对照触目惊心：

- A(TAPB+A6) 预测 0.8194、B 0.809、D3 0.783、D7 0.778、D9 0.776……**几乎全部"未形成有机械强度的膜"**
- 唯一"诚实"的是 C(TFPT+H3)：预测 0.040 并标注"模型不适合非标准官能团"

你论文里最有力的一张图，不是"双留出 PR-AUC 0.68"，而是 **"新模型对这 14 组真实实验的回测 vs 旧 GNN 当年 0.78–0.82 的虚高"**。注意 A/B 用的 TAPB 是池内常见单体，旧模型连池内都虚高——新模型有 OOF 校准，回测能直接证明它是否更诚实。这个回测我们一次都没做过。

**2. 交付错位：条件规划被弱化，而你的真实失败恰恰大多是条件问题。**

你的实验记录里写得很清楚：乙酸误加成 18M 过量、加料顺序错、溶解不完全、苯胺量与方案不符——这些是失败主因，不是单体选错。但项目这边：条件被移出主模型（D16），实验 D 的条件 PR-AUC 0.9112 疑似泄漏至今挂着未诊断，条件推荐只剩规则+案例库。**最能救你实验的交付，是我们最弱的一环。**

**3. 场景错位（部分）：我们在优化"筛选指标"，你的场景是"小集合打分"。**

你的真实场景是化学家已圈定 17 个组合、DFT 筛完，ML 给这十几个组合打分排序——不是从几千个单体里筛选。但注意：A1–A7/B1–B6/H1–H4 这些含氟特种单体**很可能落在双未见臂**（待验证：它们是否在 `models/monomer_pool.json` 池内），所以双未见 0.68 这个瓶颈对你依然是真的。区别在于解法：表征路线已全部证伪，继续刷分的价值很小，该转向**诚实化**——概率 + 不确定性 + OOD 标记一起输出。H3 那个 0.040 证明模型是能做到"知道自己不知道"的，这比 0.68 还是 0.72 重要得多。

**4. 闭环错位：主动学习不该在我们项目里重造。**

第④步的载体是 minimax RAG（shiyandiedai），不是这个项目。我们该做的是定义标准化预测输出（概率 + 不确定性 + OOD 标记 + SHAP 理由的 schema）供 RAG 消费，RAG 的实验记录（成功/失败 + 条件细节）再回流为高质量训练数据。在我们项目里自建闭环 = 重复建设。

### 重新对齐的建议优先级

1. **真实回测（论文核心图）**：把 17 个单体转成 SMILES → 查池内/池外分布 → 当前路由模型对 14 组已做实验逐一预测 → 对比旧 GNN 当年预测与真实结果 → 出"真实命中率"报告
