# COF 成膜推荐系统 · 分发说明

> 给课题组里负责转发安装包的同学。成员侧的详细操作见《用户手册》，本说明只讲"怎么发、装完做什么、数据怎么隔离"。

---

## 一、安装包怎么发

1. 安装包文件名形如 `COF成膜推荐系统-Setup-x.x.x.exe`（NSIS 安装包，Windows 64 位）。
2. 分发渠道（任选）：
   - 课题组群文件 / 网盘链接
   - U 盘拷贝（安装包较大，建议压缩包直拷）
3. **仅限课题组内部流转**，不要上传到公开渠道。
4. 转发时附上《用户手册》（`docs/用户手册.md`）或告知成员安装后看应用内指引。

## 二、成员装完要做什么（ checklist ）

发给成员时把下面这段一起转过去：

> 1. 双击安装包，按向导安装（可自定义安装目录）。
> 2. 建议把安装目录加入杀毒软件白名单（否则杀软扫描 `_internal` 会明显拖慢首次启动）。
> 3. 从开始菜单/桌面启动「COF成膜推荐系统」，**首次启动约 5–8 秒**，请耐心等待。
> 4. 如需 LLM 功能（方案卡、方案迭代问答）：打开「设置」页填入课题组提供的 base_url / api_key / model；不配置也能正常打分、排序、记录。
> 5. 8000 端口被占用不影响使用，应用会自动换端口。
> 6. 出问题先查《用户手册》FAQ，解决不了再找分发人。

## 三、数据隔离说明（重要）

- **安装包不含任何个人数据**。tree 模型与 GraphRAG 底座（graph.pkl）随包内置，所有成员拿到的是同一份"空数据"应用。
- 每位成员的用户数据独立存放于**自己电脑**的：
  `%APPDATA%\COF-Film-Recommend\`
  内容包括：收藏夹、实验记录、方案卡、LLM 配置（含 API 密钥）。
- 含义：
  - 成员之间互看不到对方的收藏与实验记录；
  - 重装/升级安装包不会覆盖该目录；
  - 换电脑迁移 = 拷贝该目录到新机器同路径；
  - LLM 密钥只在本人电脑上，请勿把 `llm_settings.local.json` 发给别人。
- 若课题组希望汇总实验记录，请走 Git 仓库的 `data/rag_export/` 契约流程（由项目维护者操作），**不要**让成员互相拷贝整个数据目录。

## 四、版本更新

- 应用已接入自动更新：安装包在 GitHub Releases（Ckx-z/kimi-tianxuan-seek-APP）**发 Release 即自动推送更新**——已安装的应用启动时自动检查新版本，用户在弹窗确认后自动下载，下载完成提示重启安装。
- 手动渠道仍有效：新版安装包发布后，成员直接覆盖安装即可；用户数据保留在 `%APPDATA%` 不受影响。
- 更新内容（模型替换、功能变更）请在群里同步一句话说明。
