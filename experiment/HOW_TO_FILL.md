# 实验反馈填表说明

你每天做完实验，如何把结果填进系统？

---

## 快速回答

打开 **`experiment/feedback_db.csv`**（中文表头），在末尾追加一行，填入必填字段。

**不要新建文件、不要修改表头、不要删旧行。**

---

## 字段说明（按重要程度）

### 必填（系统能用的最少集合）

| 字段 | 怎么填 | 示例 |
|------|--------|------|
| **方案编号** | 格式 `COF-{TAPT\|TAPB\|TFPT}-{YYYY-MM-DD}-{醛CAS}_{胺CAS}-v{N}` | `COF-TAPT-2026-07-12-1300701-03-4_443922-06-3-v1` |
| **日期** | 反馈日期 (YYYY-MM-DD) | `2026-07-12` |
| **醛CAS / 醛SMILES** | 单体库查 (CAS 必有, SMILES 后续补) | `1300701-03-4` |
| **胺CAS / 胺SMILES** | 同上 | `443922-06-3` |
| **tianxuan_预测概率** | 跑 `predict/predict_pair.py` 取 | `0.819` |
| **试剂状态** | 从 `实验/方案/冰箱试剂.xlsx` 查 | `已买` |
| **失败Class** | A / B / C / D / E / F / G (见下表) | `E` |
| **失败现象描述** | 实际观察, 50-200 字 | `液面上方玻璃壁黄褐色固体，不连续且粗糙` |
| **关联历史失败** | 上次相似的实验 ID | `COF-TAPT-2026-06-24-C-v1` |

### 强烈建议填

| 字段 | 说明 |
|------|------|
| **单一变量** | 本组唯一改的变量 |
| **科学问题** | 本组要回答的问题 |
| **根因Type** | 单体 / 条件 / 操作 (辅助标签) |
| **下一轮建议** | 你的判断或 RAG 提示 |

### 可选

| 字段 | 说明 |
|------|------|
| **PXRD文件 / FTIR文件 / SEM文件** | 文件路径, Class A/B (无产物) 可空 |
| **tianxuan_MC标准差** | MC Dropout 不确定性 |
| **版本** | v1, v2, v3... |
| **备注** | 自由文本 |

## 失败 Class A-G 速查

| Code | 含义 | PXRD |
|------|------|------|
| **A** | 完全无产物 (清液/凝胶/油) | 否 |
| **B** | 未充分反应 (大量未溶单体) | 否 |
| **C** | 无定形产物 (粉末, PXRD 馒头/无峰) | 建议 |
| **D** | 弱结晶 (PXRD 弱峰, 未达连续膜) | 建议 |
| **E** | 膜质量差 (不连续/粗糙/易碎) | 建议 |
| **F** | 膜质量中 (连续 + 部分表征达标) | 必 |
| **G** | 膜质量高 (连续 + 高结晶 + 良好 FTIR) | 必 |

打开 `experiment/feedback_db.csv`，在末尾追加：

```csv

```

**字段对齐数**：每个字段用逗号分隔，如果值里有逗号，整个值用双引号包起来。

## 进行中实验怎么处理？

日常实验（A1/A2/A5/A8/D3/D4/D7/D9 等）还在跑：

1. **不要先填 feedback_db.csv** —— 等结果稳定再说
2. **建一个自由格式笔记** 在 `experiment/notes/YYYY-MM-DD_实验X.md` 记录现场观察
3. **结论出来后** —— 把笔记压缩进 feedback_db.csv 一行

或者等实验跑完再填，**避免频繁修改 CSV**（容易出错）。

---

## 谁会用这些数据？

1. **RAG 检索器**：`bridge/search_local_pdfs.py` 用 CAS + 关键词匹配
2. **docx 生成器**：`bridge/generate_proposal.py` 引用关联失败，填到 §3 "关联历史失败"
3. **失败应对 Playbook**：`experiment/failure_playbook.md` 对每个失败给预设应对
4. **未来模型微调**：累积 50+ 条后，可以训练条件预测模型
